À propos de ActuFlash Backend

Présentation du Projet

ActuFlash est un site de presse en ligne développé dans le cadre du cours 4IPW3 (Développement Web) à l'ISFCE. Cette application web propose une expérience d'information moderne, conviviale et interactive, où les utilisateurs peuvent consulter des articles de presse, effectuer des recherches par dates, gérer leurs articles favoris et personnaliser leur expérience utilisateur.

Technologies et Architecture

Ce projet est développé selon l'architecture Model-View-Controller (MVC) à l'aide du framework AWebWiz, fourni par le professeur M. Alain Wafflard. Cette architecture permet une séparation claire des responsabilités :

  • Modèle : gestion des données et interaction avec la base de données
  • Vue : présentation et affichage des pages
  • Contrôleur : traitement des requêtes et coordination

Le backend est entièrement développé en PHP, avec une base de données MySQL pour le stockage des articles et autres informations.

3. Menu de Navigation

  • Source du menu : Fichier CSV menu.csv

  • Fonctionnalités :
    • Navigation entre les différentes pages du site
    • Affichage dynamique des catégories disponibles
    • Adaptation selon le statut d'authentification de l'utilisateur

4. Présentation des Articles

Consigne respectée : "présentation d'articles sur la page d'accueil : liste d'articles de la catégorie 'on n'est pas des pigeons' (10 max)"

  • Page d'accueil : Affichage des articles de la catégorie "On n'est pas des pigeons" (max 10)
  • Affichage :
    • Titre de l'article avec lien vers le contenu complet
    • Image associée (si disponible) ou image par défaut
    • Accroche/résumé de l'article
    • Date de publication formatée
    • Bouton pour ajouter/retirer des favoris (étoile pleine/vide)
  • Design : Grille responsive 2x2 avec effets de survol

5. Système de Recherche

Consigne respectée : "recherche d'articles par : date min max (slider or other)"

  • Type de recherche : Par plage de dates (date minimum et date maximum)
  • Interface :
    • Formulaire avec champs de date (format AAAA-MM-JJ)
    • Validation des dates saisies côté serveur
    • Affichage des résultats sous forme de liste d'articles
    • Messages d'erreur informatifs
  • Fonctionnalités :
    • Recherche automatique dès soumission du formulaire
    • Conservation des paramètres de recherche lors des actions sur les favoris
    • Gestion des erreurs et validation des formats de date
    • Vérification et inversion automatique des dates si nécessaire
    • Intégration complète avec le système de favoris

6. Gestion des Favoris

Consigne respectée : "sauvegarde du panier d'articles favoris : session"

  • Méthode de sauvegarde : Variable de session $_SESSION['favorites']
  • Fonctionnalités :
    • ✅ Ajout d'articles aux favoris depuis toutes les pages (accueil, recherche, catégories)
    • ✅ Suppression individuelle d'articles des favoris
    • ✅ Vidage complet des favoris avec confirmation
    • ✅ Page dédiée avec compteur d'articles favoris
    • ✅ Conservation de l'état et redirection intelligente
    • ✅ Indication visuelle intuitive (étoile pleine ★ / vide ☆)
    • ✅ Préservation du contexte de navigation

7. Options de Présentation

Consigne respectée : "thème : light or dark or grey" + "font : 3 font colors"

  • Thèmes disponibles :
    • Light (fond clair, texte foncé)
    • Dark (fond foncé, texte clair)
    • Grey (fond gris neutre)
  • Tailles de police :
    • Small (1rem - lecture fine)
    • Medium (1.2rem - taille standard)
    • Large (1.6rem - lecture confortable)
  • Méthode de sauvegarde : Variables de session $_SESSION['theme'] et $_SESSION['font_size']
  • Fonctionnalités :
    • Sélection via menus déroulants dans la barre de navigation
    • Application immédiate des changements
    • Conservation des paramètres de page lors des changements
    • Persistance des préférences entre les pages
    • CSS adaptatif avec classes dynamiques

8. Gestion des Catégories

  • Fonctionnalité développée : Navigation par catégories d'articles
  • Implémentation :
    • ✅ Récupération dynamique des catégories depuis la base de données
    • ✅ Affichage en grille responsive (3 par ligne)
    • ✅ Navigation vers les articles de chaque catégorie
    • ✅ Intégration avec le système de favoris
    • ✅ Design cohérent avec le reste du site
    • ✅ Messages informatifs si aucun article trouvé

9. Base de Données

  • Connexion : PDO avec MySQL/MariaDB pour une sécurité optimale
  • Tables principales :
    • t_article : articles de presse avec tous leurs détails
    • t_category : catégories d'articles
  • Interactions développées :
    • Récupération des articles pour la page d'accueil par catégorie
    • Récupération des articles par catégorie individuelle
    • Recherche d'articles par plage de dates
    • Récupération des détails d'un article spécifique
    • Récupération des articles favoris avec gestion d'erreurs
    • Requêtes optimisées avec jointures et ordre chronologique

Points d'Attention Techniques

  • Configuration serveur : Nécessite WAMP/XAMPP avec MySQL sur port 3307
  • Base de données : Doit être nommée exactement press_2024_v03
  • Fichiers requis : login.csv, menu.csv, dossier asset/static_content/
  • Sessions PHP : Nécessite que les sessions soient activées
  • Permissions : Dossier asset/ accessible en écriture

Variables de Configuration

  • DATABASE_TYPE : "SQL" (dans model.php)
  • MACHINE : "classe38" ou "home" selon l'environnement
  • AJAX_ENABLED : true/false pour activer/désactiver AJAX (préparation frontend)
  • $debug : true/false pour l'affichage des variables de session

✅ Fonctionnalités Implémentées et Testées

1. Affichage des détails d'articles (SPA) ✅

  • Consigne respectée : "on mouse over - affichage à droite de la page"
  • Comment tester :
    • Aller sur la page d'accueil
    • Survoler un article avec la souris
    • Observer l'affichage des détails dans le panneau de droite
  • Détails affichés :
    • Tous utilisateurs : titre, accroche, image, date
    • Administrateurs : + catégorie, temps de lecture, ID
  • Fichiers concernés : detail.js, detail_ajax.php

2. Gestion asynchrone des favoris (SPA) ✅

  • Consignes respectées : "gestion des favoris en variable session"
  • Comment tester :
    • Sur n'importe quelle page avec articles, cliquer sur "☆ Ajouter"
    • Observer le changement immédiat en "★ Favori" sans rechargement
    • Tester la limite de 5 favoris (message d'erreur au 6ème)
    • Aller sur la page Favoris et cliquer "Vider tous mes favoris"
    • Observer la suppression sans rechargement
  • Fonctionnalités AJAX :
    • ✅ Ajouter un article aux favoris
    • ✅ Retirer un article des favoris
    • ✅ Effacer tous les favoris
    • ✅ Limite de 5 favoris maximum
    • ✅ Notifications en temps réel
  • Fichiers concernés : articles.js, favorite_ajax.php

3. Outil de recherche multi-critères (SPA) ✅

  • Consignes respectées : "mot-clé + readtime min max (slider) + category (dropdown list)" + date(calendrier)
  • Comment tester :
    • Aller sur la page "Recherche"
    • Tester les critères de recherche suggérés (voir section "Mots-clés à tester")
    • Observer l'affichage en temps réel des critères sélectionnés
    • Cliquer sur "☆ Ajouter" dans les résultats (AJAX)
  • Layout implémenté : 3 colonnes (formulaire + bouton recherche + résultats)
  • Critères disponibles :
    • Mot-clé (titre et contenu)
    • Temps de lecture (sliders min/max synchronisés)
    • Catégorie (menu déroulant)
    • Date (menu déroulant)

4. Options de présentation (SPA) ✅

  • Consignes respectées : "thème : yellow or rose or blue" + "font : 3 font sizes"
  • Comment tester :
    • Dans la barre de navigation, utiliser les menus déroulants
    • Changer le thème (Jaune/Rose/Bleu)
    • Changer la taille de police (Petite/Moyenne/Grande)
    • Observer les changements instantanés sans rechargement
    • Naviguer entre les pages : les préférences sont conservées
  • Persistence : Préférences sauvées dans localStorage
  • Fichier concerné : theme.js

5. Système double Architecture ✅

  • Consigne respectée : "système double MPA/SPA configurable"
  • Configuration : Variable AJAX_ENABLED dans app.php
  • Comment tester :
    • Par défaut : AJAX_ENABLED = true (mode SPA)
    • Pour tester MPA : changer à false et recharger
    • Observer la différence de comportement des favoris
